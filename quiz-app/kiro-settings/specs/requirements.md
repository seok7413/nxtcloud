# Requirements — AWS 기초 개념 퀴즈 앱

## 개요

AWS 클라우드 기초 지식을 테스트하는 객관식 퀴즈 웹 앱.
100% 클라이언트사이드 정적 앱으로 동작하며, 바닐라 HTML/CSS/JS만 사용한다.

---

## 기능 요구사항

### FR-1: 퀴즈 풀기

- FR-1.1: 5개의 AWS 기초 객관식 문제를 순서대로 제시한다.
- FR-1.2: 각 문제는 질문 텍스트와 4개의 선택지로 구성된다.
- FR-1.3: 사용자가 선택지를 클릭하면 즉시 정오답을 표시한다.
- FR-1.4: 정답은 ✓ 아이콘 + 초록색, 오답은 ✗ 아이콘 + 빨간색으로 표시한다.
- FR-1.5: 정오답 판정 후 해설을 보여준다.
- FR-1.6: "다음 문제" 버튼을 눌러 다음 문제로 이동한다.

### FR-2: 점수 및 진행도

- FR-2.1: 화면 상단에 현재 점수 (맞은 수 / 전체 수)를 표시한다.
- FR-2.2: 프로그레스 바로 진행률을 시각화한다.
- FR-2.3: 현재 문제 번호를 표시한다 (예: "문제 3 / 5").

### FR-3: 결과 화면

- FR-3.1: 모든 문제를 풀면 최종 점수와 등급 메시지를 표시한다.
- FR-3.2: 등급 기준: 100% → 🏆 완벽, 60%↑ → 👍 양호, 60%↓ → 📚 학습 필요.
- FR-3.3: "전체 다시 풀기" 버튼으로 퀴즈를 초기화한다.
- FR-3.4: "틀린 문제만 다시 풀기" 버튼으로 오답만 재도전한다.

### FR-4: 오답 노트

- FR-4.1: 틀린 문제는 자동으로 오답 노트에 저장된다.
- FR-4.2: 오답 노트 탭에서 틀린 문제 목록을 확인할 수 있다.
- FR-4.3: 각 오답 카드에는 문제, 내가 선택한 답, 정답, 해설이 표시된다.
- FR-4.4: 개별 오답을 삭제할 수 있다.
- FR-4.5: 오답 노트 전체를 삭제할 수 있다 (확인 후).
- FR-4.6: 정답으로 재도전 성공 시 오답 노트에서 자동 제거된다.

### FR-5: 상태 유지

- FR-5.1: 퀴즈 진행 상태(현재 문제, 점수, 답변 기록)를 localStorage에 저장한다.
- FR-5.2: 오답 노트를 localStorage에 별도 키로 저장한다.
- FR-5.3: 페이지 새로고침 시 이전 상태를 복원한다.
- FR-5.4: localStorage 접근 실패 시 에러 없이 기본값으로 동작한다.

### FR-6: 탭 네비게이션

- FR-6.1: "퀴즈" 탭과 "오답 노트" 탭으로 구성된다.
- FR-6.2: 오답 노트 탭에 미해결 오답 수를 뱃지로 표시한다.

## 비기능 요구사항

### NFR-1: 기술 제약

- NFR-1.1: 바닐라 HTML/CSS/JS만 사용. 외부 라이브러리·CDN 금지.
- NFR-1.2: index.html / style.css / app.js 3개 파일로 구성.
- NFR-1.3: 로그인·서버·외부 API·광고 없음.

### NFR-2: 코드 품질

- NFR-2.1: innerHTML 사용 금지. textContent 또는 DOM API만 사용.
- NFR-2.2: localStorage 접근은 try/catch로 감싼다.
- NFR-2.3: 설정값은 파일 상단 상수로 선언한다.
- NFR-2.4: 전역 변수 금지. IIFE 또는 ES 모듈로 코드를 감싼다.

### NFR-3: 접근성

- NFR-3.1: 점수 표시 영역에 aria-live="polite" 적용.
- NFR-3.2: 문제 영역에 aria-live="polite" 적용.
- NFR-3.3: 답 선택 후 "다음 문제" 버튼으로 포커스 자동 이동.
- NFR-3.4: 정오답은 색상 + ✓/✗ 아이콘 병기.
- NFR-3.5: lang="ko" 지정.
- NFR-3.6: 버튼에 적절한 접근 가능한 이름 부여.

### NFR-4: UI/UX

- NFR-4.1: 반응형 레이아웃 (모바일~데스크톱).
- NFR-4.2: AWS 브랜드 컬러(#232f3e, #ff9900) 기반 디자인.
- NFR-4.3: 부드러운 전환 효과 (transition).
- NFR-4.4: 최대 폭 640px 컨테이너.
